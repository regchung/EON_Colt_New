# 派遣原則(Dispatch Principles)

即時派遣(`dispatcher.run_dispatch`)遵循的營運原則與其技術實現。
更新日期:2026-06。實現引擎:VROOM(PDPTW,shipment 模型)。

---

## 原則 1:先派固定趟、綁指定駕駛、且固定趟可共乘
- **綁指定駕駛(非綁死車號)**:固定行程於 `fixed_route.driver_name` 設「指定司機」;
  `fixed_route_match` 先用 `driver_resolve` 解出該司機**當天開哪台車**(當日輪車 > 司機預設車),
  再把固定趟釘到那台 → 指定司機換車,固定趟跟著走。實測 2026-06-22:43 筆全數解析到指定司機的車。
- **先派**:固定行程以**唯一技能硬綁**該車(`PIN_SKILL_BASE+vid`),`priority=80`(一般單 50)→ 優先排入,
  且只有該車能接(別車無此技能)。
- **邊界**:若指定司機當天查無車(`unresolved_rules`),該固定趟目前**退回一般派遣**(交其他可用車);
  若要改為「無指定司機則保留不派」,需另開選項。
- **可共乘**:固定趟訂單強制 `excl=1`(可與他人併車),**不受「共乘需同意」限制**——
  與一般單「未同意→獨佔整車(`excl=EXCL_CAP`)」不同。
- 程式:`dispatcher.py` excl 判定加 `if o.id in fixed_pins: excl = 1`。

## 原則 2:同一區域盡量放同一台車(軟性)
- 由 VROOM **最小化總車程**的目標函數自然達成:鄰近上/下車點會被聚到同一條路線/同一台車。
- **刻意做成軟性**(非硬牆):需要時仍可跨區調度,避免切斷高效共乘。
- 可調旋鈕:`zone_affinity`(區域親和,弱訊號)為未來要「加強」同區傾向時的權重來源,
  預設不硬接(見 BACKLOG「區域親和接進批次排班」)。

## 原則 3:訂單間 ≤ 1 小時車程、同一行政區優先(軟性)
- 同樣由「最小化總車程」目標達成:長腿(>1h)與跨行政區的接續會被自然懲罰、儘量避免。
- **刻意不設硬上限**:實測顯示收太緊會使未派上升、用車增加(與省車目標相反,
  見 `open-decisions.md` D1 單日 what-if)。保持軟性,讓引擎在效率與覆蓋間取最佳解。

## 原則 4:是否需福祉車「只看匯入時的車型文字」
- **判斷時機與依據**:**匯入時**看「車型」欄位的值——值帶有「福祉車」文字(含「福祉」)→
  此單**必須由福祉車接單**;其餘任何值(一般車/復康巴士/輪椅/空白…)→ **不限車種**。
- 落地:匯入 `_parse_vehicle_type` 把含「福祉/福祉車/welfare」的車型存為 `vehicle_type='welfare'`,
  其餘存 `'normal'`;`vehicle_type` **純由車型欄位決定**,不受「輪椅」欄位影響。
- 派遣:`dispatcher._is_welfare(o) = (o.vehicle_type == 'welfare')`,**不再**以 `need_wheelchair` 判定。
- 機制:福祉車帶技能 `{1}`;需福祉的單帶技能 `{1}` → 只能上福祉車;非福祉單無此技能 → 任意車
  (福祉車為能力超集,閒時可兼接一般單)。

## 原則 5:司機出勤緩衝(發車提早、收車跨過)
- **政策**:服務時段 06:00–18:00(乘客上車限此區間不變),但**司機/車輛可用時段 = 05:30–18:30**
  ——可提早 30 分發車去接 06:00 早單;「18:30 前已上車」的在途趟次**可跨過 18:30 完成**(保留完成緩衝)。
- 落地:設定 `driver_margin_min`(預設 30,可於參數頁調整)。
  車輛時間窗 = `[服務起 − margin, 服務迄 + 完成緩衝]`;乘客上車窗仍限服務時段(`out_of_service` 不變)。
- 程式:`dispatcher` `veh_day_start = day_start - margin`;`veh_day_end = day_end + completion_buffer`。
- 實測(2026-06-22):05:30 發車讓 06:00 桃園早單排入,未派 5→4、已派 202→203(用 38 車)。

## 不變的硬約束(護欄)
時間窗(上車 `[T, T+window]`、服務時段 06–18)、座位容量、8h 工時上限、
進行中行程鎖原車、固定行程釘指定車 —— 皆為**硬約束**,不因上述軟性原則放寬。
