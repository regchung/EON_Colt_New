# 研究領域對應:哪些學術知識能協助 SmartCar

> 用途:盤點與本案**實際組件 / 開放問題**對得上的學術領域,標註價值評級,作為
> 「何時該借重學術方法、何時該忍住別過度工程」的決策參考。
> 原則延續本案策略:**精簡、別上重模型**(類神經 / TimesFM 已實測對本案效益低,維持否決)。
> 評估日期:2026-06。格式:領域 → 對應 SmartCar 組件 → 能改善什麼 → 評級。

---

## 🟢 高價值(直接對應核心、本案尚未吃滿)

### 1. 作業研究 / 組合最佳化(Operations Research / Combinatorial Optimization)— 本案心臟
- **對應**:VROOM 派遣(PDPTW、車容量、時間窗、技能鎖定、首末站錨定)。
- **能改善**:VRP 變體建模正確性、軟/硬約束設計、求解品質上限判斷。
  「區域親和當軟偏好」「上車窗 30/45/60 取捨」本質都是 OR 的目標函數權衡。
- **評級**:最該借重——決定「換不換引擎、約束怎麼擺」。引擎選型見
  `eval-pyvrp-vs-vroom.md` / `eval-docling-tenancy-timefold.md`。

### 2. 統計學 / 計量經濟(Statistics / Econometrics)— 對比與決策的根基
- **對應**:人工 vs 自動對比、時間窗敏感度、NT$ 換算、未派原因聚合。
- **能改善**:**因果推論 / 反事實估計**(「若共乘、若放寬時間窗會省多少」即反事實);
  信賴區間與顯著性,避免把雜訊當訊號(「駕駛熟區訊號偏弱 11–21%」即顯著性的直覺版)。
- **評級**:直接提升報告可信度——對賣車隊關鍵,數字要站得住。

### 3. 時間序列預測(Time Series Forecasting)— 已用輕量基線,理論在此
- **對應**:需求預測(weekday 季節基線 → 各日趟次 / 建議排車數)。
- **能改善**:季節性分解(STL)、週期性檢定、何時該升級多序列模型的**判準**。
- **評級**:已正確判斷「資料短、週循環為主 → 輕量基線足夠」。升級條件未達,維持現狀。

## 🟠 中價值(特定子問題有用,別全盤上)

### 4. 圖論 / 匹配理論(Graph Theory / Matching)
- **對應**:共乘推薦、常態共乘對挖掘(57 組反覆同行)、固定行程指派。
- **能改善**:共乘配對 = 最大權重匹配 / 集合覆蓋;固定行程 = 二分圖匹配。
  把啟發式換成有理論保證的匹配,可能再擠出共乘增益。
- **評級**:最可能直接變現的子題之一(把共乘配對正規化為最大權重匹配)。

### 5. 機器學習 — 限「弱訊號排序」(非深度學習)
- **對應**:徵詢成功率學習(BACKLOG #1)、駕駛忠誠度、區域親和。
- **能改善**:「依情境排序 / 預測機率」是**邏輯回歸 / 梯度提升樹(GBDT)**的甜區。
  等 `pool-consent` 累積真實結果,用 GBDT 預測「這通徵詢成功率」排序撥打順序。
- **評級**:可做,但**守住底線**——類神經 / TimesFM 已驗證不划算,別回頭。

### 6. 強化學習(Reinforcement Learning)— 概念有用,落地要忍住
- **對應**:徵詢順序、動態重排策略。
- **能改善**:**contextual bandit(情境式多臂老虎機)**是「有限預算下先打誰」的正規框架。
- **評級**:需大量線上回饋,現階段資料不足 → **理論先記著,別實作**。

## 🔵 商業化硬需求對應的領域(合規門票,非性能引擎)

### 7. 隱私工程 / 資料庫安全(Privacy Engineering)
- **對應**:多租戶隔離(Postgres RLS)、PII 合規(Docling 本地抽取 + 去識別化中介層)。
- **能改善**:去識別化、k-匿名、RLS 形式化。送 Claude 前的去識別化即此領域標準做法。
- **評級**:收費前必修。見 `eval-docling-tenancy-timefold.md`。

### 8. 自然語言處理 — 資訊抽取(NLP / Information Extraction)
- **對應**:AI 文件匯入、出勤文字解析、調度助理。
- **能改善**:結構化抽取(NER + schema)、本地小模型抽取(配合 Docling 走地端),
  減少對 Claude 的依賴與 PII 外送。
- **評級**:與隱私工程綁定,商業化階段的支撐。

### 9. 地理資訊科學(GIScience)
- **對應**:Map8 地理編碼、OSRM 矩陣、地址正規化、地址簿歸一門牌。
- **能改善**:地址匹配 / geocoding 容錯、record linkage、空間索引。
- **評級**:已有地址簿快取機制;進一步容錯可降低未編碼率。

## 一句話總結
> 對本案最划算的三個學術支點:**作業研究(把派遣約束建對)+ 統計/計量(把對比數字做到
> 能說服車隊)+ 弱訊號 ML / 匹配理論(徵詢排序、共乘配對)**。
> 其餘(隱私工程、IE、GIScience)是**商業化的合規與資料品質門票**,不是性能引擎。

## 最可能直接變現的兩個切入點(候選,未實作)
1. **共乘配對正規化為最大權重匹配**(圖論)——可能再擠出共乘增益。
2. **對比報告補上統計顯著性 / 信賴區間**(統計)——提升對車隊的說服力。
